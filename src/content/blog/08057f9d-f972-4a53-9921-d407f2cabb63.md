本文整理了一套最常用、最核心的 Git 指令及其含义，适合用于快速入门或作为日常参考。

## 一、拉取项目

使用 “git clone 项目地址” 可以把远程仓库的代码克隆到本地。

## 二、查看远程仓库信息

通过 “git remote -v” 可以查看当前项目关联的远程仓库列表。 典型输出如下：

origin git@github.com:user/repo.git (fetch) origin git@github.com:user/repo.git (push)

其中：

*   origin：远程仓库名称
    
*   git@github.com:user/repo.git：远程仓库地址
    
*   fetch / push：分别对应拉取与推送时使用的地址
    

## 三、添加远程仓库

如果你的本地项目还没有关联任何远程仓库，可以使用 “git remote add 名称 远程仓库地址"来手动添加远程仓库。

## 四、拉取远程分支（可设置上游）

使用 “git pull -u 远程仓库名称 分支名” 可以从指定的远程仓库拉取代码。 其中 -u 参数的作用是将本地分支与远程分支建立关联，也就是设置上游分支。

设置上游分支后，以后就可以直接使用 “git pull” 和 “git push”，不需要再输入远程仓库名称和分支名。

## 五、推送代码到远程仓库

使用 “git push 远程仓库名称 分支名” 可将本地代码推送到远程仓库。 如果当前分支已经设置了上游分支，那么只需要输入 “git push” 即可完成推送。

## 六、查看分支

查看本地分支列表： 使用 “git branch” 可以查看全部本地分支。

查看分支的详细信息： 使用 “git branch -vv” 可以看到每个分支对应的远程分支、提交信息以及追踪关系。
