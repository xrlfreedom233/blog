> 本文面向完全没有 Linux 经验的新手，帮你从选发行版到装软件，少走弯路。

* * *

## 一、先选发行版

Linux 不是一个系统，而是一大家子系统。对于新手，只需要关注以下几个：

发行版

特点

推荐人群

**Ubuntu**

社区最大、文档最全、最好搜到答案

所有新手首选

**Fedora**

软件版本新、技术前沿、稳定性好

想用新技术的开发者

**Linux Mint**

基于 Ubuntu、界面最像 Windows

Windows 迁移用户

**EndeavourOS**

基于 Arch、软件最全、滚动更新

有一点基础想进阶的用户

**结论：第一次用 Linux，装 Ubuntu，没有之一。**

遇到问题随便一搜都有答案，其他发行版做不到这一点。

* * *

## 二、安装系统

### 准备工作

1.  下载发行版 ISO 镜像（官网下载，国内可用清华源加速）
    
2.  准备一个 8GB 以上的 U 盘
    
3.  下载 [Rufus](https://rufus.ie/)（Windows）或 [balenaEtcher](https://etcher.balena.io/)（跨平台）将 ISO 写入 U 盘
    

### 安装步骤

1.  重启电脑，按 F2 / F12 / Del 进入 BIOS（不同品牌按键不同）
    
2.  将启动顺序改为 U 盘优先
    
3.  保存退出，进入 Linux 安装界面
    
4.  跟着图形安装器一步步走，选语言、时区、磁盘分区
    
5.  设置用户名和密码，等待安装完成
    

> **磁盘分区建议**：新手直接选"自动分区"或"使用整个磁盘"，不要手动折腾。如果是双系统，务必先备份 Windows 数据。

## 三、认识包管理器

这是 Linux 和 Windows 最大的不同之一。Linux 装软件不需要去官网找安装包，用**包管理器**就能搞定。

不同发行版用不同的包管理器：

发行版

包管理器

常用命令

Ubuntu / Mint

apt

`sudo apt install 软件名`

Fedora

dnf

`sudo dnf install 软件名`

Arch / EndeavourOS

pacman

`sudo pacman -S 软件名`

以 Ubuntu 为例，常用操作：



```
bash
sudo apt update              # 更新软件源列表（先运行这个）
sudo apt install 软件名       # 安装软件
sudo apt remove 软件名        # 卸载软件
sudo apt upgrade             # 更新所有已安装软件
```

* * *

## 四、装软件的几种方式

### 1\. apt / dnf / pacman（首选）

官方源里有的软件，优先用包管理器装，干净、好卸载、自动更新。
```
bash
# Ubuntu 示例
sudo apt install git curl wget vim
```

### 2\. DEB / RPM 包

去官网下载 `.deb`（Ubuntu）或 `.rpm`（Fedora）安装包，本地安装：
```
bash
# Ubuntu
sudo apt install ./软件名.deb

# Fedora
sudo dnf install ./软件名.rpm
```

### 3\. Flatpak（推荐用于桌面应用）

跨发行版的通用包格式，软件运行在沙盒里，更安全。[Flathub](https://flathub.org/) 上有大量应用。
```
bash
# Ubuntu 安装 Flatpak 支持
sudo apt install flatpak
flatpak remote-add --if-not-exists flathub https://flathub.org/repo/flathub.flatpakrepo

# 安装软件
flatpak install flathub 软件名
```

### 4\. Snap（能用但不推荐）

Ubuntu 官方推的格式，但启动慢、体积大，很多软件被悄悄替换成了 Snap 版（比如 Firefox）。能用，但优先选前几种方式。

### 5\. AppImage（免安装）

下载后直接运行，不需要安装：
```
bash
chmod +x 软件.AppImage   # 赋予执行权限
./软件.AppImage           # 运行
```

### 优先顺序总结
```
apt/dnf/pacman  >  Flatpak  >  AppImage  >  Snap
```

## 五、必须掌握的基础命令

不需要背，用到时查，用多了自然记住。

### 文件操作
```
bash
ls          # 查看当前目录文件
ls -la      # 查看详细信息（含隐藏文件）
cd 目录名   # 进入目录
cd ..       # 返回上一级
cd ~        # 回到主目录
pwd         # 查看当前所在路径

cp 源文件 目标    # 复制文件
mv 源文件 目标    # 移动/重命名文件
rm 文件名         # 删除文件
rm -rf 目录名     # 删除目录（谨慎！）
mkdir 目录名      # 新建目录
```

### 查看内容
```
bash
cat 文件名        # 查看文件内容
less 文件名       # 分页查看（q 退出）
head -n 10 文件   # 查看前 10 行
tail -f 文件      # 实时查看文件末尾（常用于看日志）
```

### 权限与管理
```
bash
sudo 命令          # 以管理员权限运行
chmod +x 文件名    # 赋予执行权限
chown 用户 文件    # 修改文件所有者
```

### 进程管理
```
bash
ps aux             # 查看所有进程
kill 进程ID        # 终止进程
htop               # 交互式进程管理（需要先安装：sudo apt install htop）
```

### 其他常用
```
bash
grep "关键词" 文件   # 在文件中搜索
find / -name "文件名" # 全盘搜索文件
df -h               # 查看磁盘使用情况
free -h             # 查看内存使用情况
history             # 查看命令历史
man 命令名          # 查看命令的帮助文档
```

## 六、几个新手常见坑

### 坑 1：用 rm -rf 删错了东西

Linux 没有回收站，`rm -rf` 删掉的东西找不回来。 **养成习惯：删除前先用** `ls` **确认路径。**

### 坑 2：忘记 sudo

很多命令需要管理员权限，报"Permission denied"时加上 `sudo` 再试。

### 坑 3：安装软件前没有 apt update

软件源缓存过期会导致找不到包或安装旧版本。养成习惯，`apt install` 前先 `apt update`。

### 坑 4：修改系统文件没有备份

改配置文件前先备份：
```
bash
cp /etc/配置文件 /etc/配置文件.bak
```

### 坑 5：不知道去哪搜答案

*   [**Arch Wiki**](https://wiki.archlinux.org/)：Linux 最好的文档，即使不用 Arch 也能参考
    
*   [**Stack Overflow**](https://stackoverflow.com/) / [**Ask Ubuntu**](https://askubuntu.com/)：遇到报错直接把错误信息粘贴过去搜

## 七、开发环境快速配置（选看）

如果你用 Linux 做开发，以下是常用环境的安装方式。

### Git
```
bash
sudo apt install git
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

### Node.js
```
bash
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo bash -
sudo apt install nodejs
```

### Python

Ubuntu 自带 Python 3，直接用。pip 安装包：
```
bash
sudo apt install python3-pip
pip3 install 包名
```

### JetBrains IDE（如 IntelliJ IDEA）

推荐用 JetBrains Toolbox 管理：

1.  去 [JetBrains 官网](https://www.jetbrains.com/toolbox-app/) 下载 Toolbox tar.gz
    
2.  解压并运行：
    
```
bash
tar -xzf jetbrains-toolbox-*.tar.gz
cd jetbrains-toolbox-*/bin/
./jetbrains-toolbox
```

3.  在 Toolbox 图形界面里点击安装你需要的 IDE
    

### Docker
```
bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER   # 让当前用户免 sudo 使用 docker
# 重新登录后生效
```

## 总结

Linux 上手曲线确实存在，但度过最初的陌生期之后，你会发现它在开发、效率上远超 Windows。

核心建议：

1.  **从 Ubuntu 开始**，不要一上来就折腾 Arch
    
2.  **遇到报错不要慌**，把错误信息原文搜索，99% 都有答案
    
3.  **多用终端**，图形界面能做的事终端都能做，而且往往更快
    
4.  **不要乱 sudo**，明白自己在做什么再执行
    

祝开荒顺利。
